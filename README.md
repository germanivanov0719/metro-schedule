# Метро СПб · Режим работы станций

Небольшое PWA-приложение: показывает время работы вестибюлей, первые и
последние поезда метрополитена Санкт-Петербурга в удобном для телефона виде.
Полностью статическое, разворачивается на GitHub Pages, устанавливается как
приложение и работает офлайн.

Источник данных — официальная страница расписания
<https://metro.spb.ru/rejimrabotystancii.html>.

## Как это работает

Никаких данных на сервере не хранится. При каждом запуске приложение
**скачивает официальную страницу расписания и парсит её прямо в браузере**
тем же `parser.js`. Удачный результат кладётся в `localStorage` и используется:

- мгновенно при следующих запусках (показывается сразу, пока идёт обновление);
- как резерв, если сеть недоступна или официальный сайт не открывается.

Так как у `metro.spb.ru` нет CORS-заголовков, запрос с GitHub Pages идёт через
публичные бесплатные CORS-прокси (allorigins, codetabs, thingproxy) — они
перебираются по очереди, пока какой-то не вернёт корректную страницу. Пока
данных ещё нет, показывается индикатор загрузки.

Парсер намеренно «гибкий»: он не привязан к жёстким позициям колонок, а
разворачивает `rowspan`/`colspan` в матрицу и определяет роль колонок по
ключевым словам в заголовках, поэтому мелкие правки вёрстки сайта его не ломают.

### Возможности интерфейса

- Светлая и тёмная темы; по умолчанию — как в системе. Переключатель внизу.
- Поиск по станциям и вестибюлям, фильтр по линиям в их фирменных цветах.
- Часы и текущая чётность дня; для первых поездов подсвечивается сегодняшняя
  колонка (чёт/нечёт), нерелевантная слегка затемняется.
- Живой обратный отсчёт до последнего поезда и индикация «вход открыт/закрыт».
- Шрифты PT Serif и PT Sans (Google Fonts) кэшируются service worker'ом для
  офлайна; есть системные fallback-шрифты.

## Состав

| Файл | Назначение |
|------|-----------|
| `index.html` | Оболочка приложения |
| `styles.css` | Светлая/тёмная темы, оформление |
| `app.js` | Логика: загрузка с сайта, кэш, рендер, поиск, темы, часы |
| `parser.js` | Гибкий парсер таблиц официальной страницы |
| `sw.js` | Service worker: кэш оболочки и шрифтов для офлайна |
| `manifest.webmanifest` | Манифест PWA |
| `icons/` | Иконки приложения |
| `gen-data.mjs` | *(необязательно)* генератор статического снимка `data.json` в Node |
| `make-icons.py` | *(необязательно)* генератор иконок (Python + Pillow) |

## Развёртывание на GitHub Pages

1. Создайте репозиторий и положите туда все файлы (включая папку `icons/`).
2. **Settings → Pages**.
3. В *Build and deployment* выберите **Deploy from a branch**, ветку (`main`)
   и папку `/ (root)`. Сохраните.
4. Через минуту приложение будет доступно по адресу
   `https://<логин>.github.io/<репозиторий>/`.

Все пути относительные (`./…`), поэтому приложение одинаково работает и в корне
домена, и в подпапке проекта. HTTPS у Pages включён по умолчанию — это нужно
для service worker и установки PWA.

### Установка как приложение

Откройте сайт в браузере телефона → «Добавить на главный экран»
(Android/Chrome предложит автоматически; на iOS/Safari — через «Поделиться»).
После первой загрузки приложение работает без сети.

## Локальный запуск

ES-модули и `fetch` требуют http(s), через `file://` не работает:

```bash
python3 -m http.server 8000
# откройте http://localhost:8000/
```

## Необязательные инструменты

Статический снимок расписания (если захотите положить `data.json` рядом —
приложению он не нужен, но может пригодиться для архива):

```bash
node gen-data.mjs path/to/saved-page.html
```

Перегенерация иконок (нужен Python с Pillow):

```bash
python3 make-icons.py
```

## Оговорка

Приложение неофициальное. Данные могут отличаться от актуальных — перед
поездкой сверяйтесь с официальным сайтом метрополитена.
