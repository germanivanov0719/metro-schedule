# Петербургский метрополитен · режим станций

PWA-приложение: время работы вестибюлей, первые и последние поезда метро
Санкт-Петербурга в удобном для телефона виде. Статическое, разворачивается на
GitHub Pages, ставится как приложение и работает офлайн.

Источник — официальная страница расписания
<https://metro.spb.ru/rejimrabotystancii.html>.

## Как берутся данные

Три источника, по свежести:

1. **`data.json`** — снимок расписания в репозитории. Даёт мгновенный показ и
   офлайн с первого запуска. Обновляется автоматически через GitHub Actions
   (workflow `.github/workflows/update-data.yml`, ежедневно).
2. **localStorage-кэш** — результат последнего удачного живого запроса.
3. **Живой запрос** на официальный сайт — выполняется при каждом запуске; HTML
   парсится прямо в браузере тем же `parser.js`, результат кладётся в кэш.

У `metro.spb.ru` нет CORS-заголовков, поэтому из браузера запрос идёт через
публичные бесплатные прокси (allorigins, codetabs, thingproxy) — перебираются по
очереди. Пока данные грузятся, показывается индикатор; сверху всегда видно время
последнего обновления и статус «обновление…».

Парсер устойчив к мелким правкам вёрстки: он разворачивает `rowspan`/`colspan`
в матрицу и определяет роль колонок по ключевым словам заголовков.

## Возможности

- Светлая/тёмная темы, по умолчанию — системная; переключатель внизу.
- Поиск по станциям и вестибюлям; фильтр по линиям значками-«М» в цветах линий.
- Названия станций — в цвете линии; под ними кликабельные пересадки (М4 Спасская
  и т. п.), переход сразу открывает нужную станцию.
- По каждому направлению — отсчёт до первого/последнего поезда с цветовой
  индикацией (красный «поезд ушёл», жёлтый < 30 мин, зелёный иначе); стрелки
  направлений «А ↔ Б» и «→ Станция».
- Подсветка сегодняшней чётности (чёт/нечёт), нерелевантная колонка приглушена.
- Базовые госпраздники: в «ночные» праздники — заметка о возможном круглосуточном
  режиме со ссылкой на сайт метро.
- Разовое напоминание про закрытие пересадок в 00:15 (потом — свёрнутым внизу).
- Шрифты PT Serif, PT Sans и Roboto (для номеров линий) с Google Fonts,
  кэшируются service worker'ом для офлайна; есть системные fallback'и.

## Состав

| Файл | Назначение |
|------|-----------|
| `index.html`, `styles.css`, `app.js` | Оболочка, оформление, логика |
| `parser.js` | Гибкий парсер таблиц официальной страницы |
| `data.json` | Снимок расписания (обновляется GitHub Actions) |
| `sw.js` | Service worker: кэш оболочки, снимка и шрифтов |
| `manifest.webmanifest`, `icons/` | Манифест и иконки PWA |
| `.github/workflows/update-data.yml` | Автообновление `data.json` |
| `gen-data.mjs` | Генератор снимка (локально или из сети) |
| `make-icons.py` | Генератор иконок (Python + Pillow) |

## Развёртывание на GitHub Pages

1. Залейте все файлы в репозиторий (включая `icons/` и `.github/`).
2. **Settings → Pages → Deploy from a branch**, ветка `main`, папка `/ (root)`.
3. Откройте `https://<логин>.github.io/<репозиторий>/`.

Пути относительные, поэтому работает и в корне, и в подпапке. HTTPS у Pages
включён по умолчанию — нужно для service worker и установки PWA.

**Автообновление расписания.** В *Settings → Actions → General* разрешите
*Read and write permissions* для GITHUB_TOKEN. Workflow можно запустить вручную
из вкладки **Actions → Обновление снимка расписания → Run workflow**. Если у
официального сайта серверная загрузка не пройдёт, снимок просто останется
прежним, а приложение всё равно обновит данные живым запросом из браузера.

### Установка как приложение

Откройте сайт в браузере телефона → «Добавить на главный экран». После первой
загрузки приложение работает без сети.

## Локальный запуск

```bash
python3 -m http.server 8000   # http://localhost:8000/
```

## Инструменты

```bash
node gen-data.mjs                         # из встроенной сохранённой копии
node gen-data.mjs https://metro.spb.ru/rejimrabotystancii.html   # из сети
python3 make-icons.py                     # перегенерировать иконки (нужен Pillow)
```

## Оговорка

Приложение неофициальное; данные могут отличаться от актуальных. Перед поездкой
сверяйтесь с официальным сайтом метрополитена.
